Accessibility

Browser compatibility

When using media queries to make a web-site responsive add respond.js to make them compatible with Internet Explorer 8 and under. To make HTML5 elements compatible with most common browsers that do not support it, use html5shiv. Links to github provided below:

  • Github html5shiv
  • Github respond

Semantics

With HTML5 elements your code will be much easier to read and understand due to the consistency they bring into the code. This will help both developers and search-engines to make sense of your site. For example, see the HTML for this site and the use of header, nav, main, section and article and how they give meaning to the page.

Images

To make a website clear for a user even if a site does not load images or if using a screen reader, the attribute ‘alt’ can be used to explain what is on the image. The attribute ‘title’ gives a title and displays it when the mouse is hovering over a picture. These gives more understanding of the page. Try hovering over an image on the Home page.
